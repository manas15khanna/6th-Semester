Problem Description

 

Your team at a leading technology company is launching a cutting-edge networking product. To generate buzz, you're running a targeted social media campaign. You have a limited marketing budget to send premium trial units to a select group of "seed" influencers on a professional network.

Each potential influencer has a different cost to engage (e.g., the cost of the trial unit or compensation). Furthermore, not all users in the network are equally valuable to the campaign; each user has an associated market value based on their role and influence (e.g., a 'Lead Network Architect' is more valuable than an 'Intern').

Your task is to develop an algorithm that selects the optimal set of seed influencers. The "optimal set" is the one that maximizes the total market value of all unique users reached by the campaign, without exceeding the given budget. Influence spreads from the seeds to their friends, their friends-of-friends, and so on, up to a specified depth d.

A brute-force approach that checks every possible combination of seeds is computationally infeasible. You will need to devise a more clever heuristic to find a high-quality solution.

Input

 

  • connections: A list of pairs representing the connections between users. User IDs are integers. Example: [(1, 2), (2, 3)].
  • costs: A dictionary (or map) where keys are integer user IDs and values are their corresponding costs to be selected as a seed. Users not in this map cannot be chosen as seeds. Example: {1: 50, 2: 60}.
  • values: A dictionary (or map) where keys are integer user IDs and values are their market value. Example: {1: 10, 2: 15}.
  • budget: A single integer representing the total available budget for the campaign.
  • d: A single integer representing the maximum connection depth (or degrees of separation) for the campaign's reach. d=1 means only direct friends are reached.

Input Format

 

The input should be read from standard input (stdin) in the following format:

no_of_users
no_of_connections
connection_pairs (one pair per line: user1 user2)
no_of_potential_seeds
potential_seeds_with_costs (one pair per line: user_id cost)
values (n space-separated integers representing value for each user from 1 to n) 
budget
depth

Detailed Format:

  • Line 1: n (number of users, users are numbered from 1 to n)
  • Line 2: num_connections (number of connections)
  • Next num_connections lines: Two integers u v representing a connection between user u and user v
  • Line 3 + num_connections: num_seeds (number of potential seeds)
  • Next num_seeds lines: Two integers user_id cost representing potential seed and its cost
  • Line 4 + num_connections + num_seeds: n space-separated integers representing the value of each user (value[i] is value for user i+1)
  • Line 5 + num_connections + num_seeds: budget (total available budget)
  • Line 6 + num_connections + num_seeds: depth (maximum influence depth)

  2. The total market value reached by this set of seeds.
  3. The total cost of the chosen seeds.

Examples

 

Example 1: The High-Efficiency Choice

 

This example demonstrates a straightforward case where one candidate is clearly the most efficient first choice.

Input:

  • connections: [(1, 2), (1, 3), (4, 5)]
  • costs: {1: 50, 2: 20, 3: 20, 4: 60, 5: 20}
  • values: {1: 10, 2: 20, 3: 30, 4: 5, 5: 5}
  • budget: 100
  • d: 1

Network Structure:

User 1 (Cost: 50, Value: 10) --- User 2 (Cost: 20, Value: 20)
    |
User 3 (Cost: 20, Value: 30)

User 4 (Cost: 60, Value: 5) --- User 5 (Cost: 20, Value: 5)

Output:

  • Chosen Seeds: [1]
  • Total Value: 60
  • Total Cost: 50

Explanation: The algorithm must decide which seed to pick first. Let's analyze the initial efficiency (total_value / cost) of the potential seeds.

  1. Choose Seed 1: Cost: 50. Reach (d=1): {1, 2, 3}. Value: 10 + 20 + 30 = 60. Efficiency: 60 / 50 = 1.2.
  2. Choose Seed 4: Cost: 60. Reach (d=1): {4, 5}. Value: 5 + 5 = 10. Efficiency: 10 / 60 ≈ 0.17.

Seed 1 offers the highest value-per-cost. We select User 1. The cost is 50, and the remaining budget is 50. The users reached are now {1, 2, 3}. Next, the algorithm checks if it can add another seed. No other affordable seed can add any new value, so the algorithm stops.

Example 2: The Marginal Gain Choice

 

This example shows how the algorithm must account for overlapping reach when making its second choice.

Input:

  • connections: [(1, 2), (1, 3), (4, 3), (4, 5)]
  • costs: {1: 60, 4: 70}
  • values: {1: 5, 2: 10, 3: 20, 4: 5, 5: 30}
  • budget: 150
  • d: 1

Network Structure:

User 1 (Cost: 60, Value: 5) --- User 2 (Value: 10)
    |                               |
User 3 (Value: 20) ----------- User 4 (Cost: 70, Value: 5)
                                    |
                               User 5 (Value: 30)

Output:

  • Chosen Seeds: [4, 1]
  • Total Value: 70
  • Total Cost: 130

Explanation: Iteration 1: Find the first seed.

  1. Candidate 1: Cost=60. Reach={1, 2, 3}. Value=5+10+20=35. Efficiency=35/60 ≈ 0.58.
  2. Candidate 4: Cost=70. Reach={3, 4, 5}. Value=20+5+30=55. Efficiency=55/70 ≈ 0.78.

Seed 4 is more efficient and is chosen first.

  • State: Seeds=[4], Cost=70, Reached={3, 4, 5}, Value=55, Budget Left=80.

Iteration 2: Find the second seed. The only affordable candidate is User 1.

  • Candidate 1 (Marginal Gain):
    • Cost: 60.
    • Full Reach: {1, 2, 3}.
    • New Users added: {1, 2, 3} - {3, 4, 5} = {1, 2}.
    • Marginal Value: values[1] + values[2] = 5 + 10 = 15.
    • Marginal Efficiency: 15 / 60 = 0.25.

Since this choice adds positive value and is within budget, we select User 1.

  • Final State: Seeds=[4, 1], Cost=70+60=130, Value=55+15=70.

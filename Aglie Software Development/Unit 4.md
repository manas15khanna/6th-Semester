# Agile Testing Techniques 
Agile testing is a type of software testing that follows the principles of ASD to test the software application
## Agile Testing Principles
- Continuous Feedback 
- Parallel Testing 
- Team Involvement 
- Lightweight Documentation 
- Clean Code 
- Cost and Feedback 
- Customer Focus 
- Test Driven Approach 
## Features of Agile Testing 
1. Simple Approach 
2. Continuous Improvement 
3. Self Organized 
4. Testers enjoy their work 
5. Encourage constant communication 
6. Constant Feedback 
# ATM (Agile Testing Methodology)
1. **TDD (Test Driven Development):** In TDD, test cases are written before writing the actual code 
	1. Writing a unit test 
	2. Coding to pass the test 
	3. Refactoring the Code 
2. **BDD(Behavior Driven Development):** BDD is about testing and understanding how users interact with the features 
	It focuses on creating features based on user behavior 
3. **Exploratory Testing:** Exploratory testing is a type of software testing in which the tester is free to select any possible methodology to test the software 
4. **ATDD(Acceptance Test Driven Development):** ATDD involves the customers, developers, and the testers working together to define the requirements and challenges before coding begins 
5. **XP(Extreme Programming):** Delivers High Quality Software to satisfy the customers completely 
6. **SBT(Session Based Testing):** This method involves structured time limited testing sessions. (45-90 minutes)
7. **DSDM(Dynamic Software Development Method):** It is an agile framework for delivering the projects 
8. **CM(Crystal Methodologies):** Focuses on the people involved in a project rather than processes or tools 
# Agile Testing Strategies 
1. **Iteration 0:** Initiate the project by setting up the testing environment, identifying the team, and defining the testing tools and scope.
2. **Construction Iterations:** Deliver a working system through continuous development and testing.
    - **Confirmatory Testing:** Verifies that the software meets the specific requirements and stakeholder intent.
    - **Investigative Testing:** Uncovers hidden defects, integration issues, or non-functional risks like security and performance.
3. **Release (End Game):** The transition phase where final system testing, user training, and documentation are completed to ensure the product is ready for deployment.
4. **Final Bug Removal:** Perform intensive "bug hunting" and stabilization during the Release phase to ensure the system is error-free before the official launch.
5. **Production:** The final stage involving the live operation of the software, providing ongoing support, and gathering feedback for future updates.
